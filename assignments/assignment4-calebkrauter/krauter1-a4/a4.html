Web Service 1: public etailer site for posting cpus

Service Endpoint HTTP METHOD Function/Expected Output
http://localhost:3000/:cpuBrand/:price GET Retrieve all cpus in cpus table that share the same brand and price. (Consumer searches cpu filtered by brand and price.)
http://localhost:3000/:cpuBrand/:price/:coreCount GET Retrieve all cpus in cpus table that share the same brand, price, and core count.(Consumer searches cpu filtered by brand and price and core count.)
http://localhost:3000/:cpuBrand/:price/:coreCount/:serialNumber GET Retrieve all data on cpu a specific cpu. (Consumer to search by serial number to review their specific listing.)
http://localhost:3000/:serialNumber POST Insert a new cpu by serial number (Consumer uploads cpu to sell by serial number to help ensure that the unique serial number is listed acurately.)
http://localhost:3000/:cpuBrand/:price/:coreCount/:serialNumber PUT Update an existing cpu by serialNumber (Consumer updates listing by brand, price, core count and serial number to verify what listing is being updated.)
http://localhost:3000/:cpuBrand/:price/:coreCount/:serialNumber DELETE Delete a cpu by serial number (Consumer deletes listing by serial number.)


Web Service 2: public etailer site from the Admin's role

Service Endpoint HTTP METHOD Function/Expected Output
http://localhost:3000/ GET Retrieve all data on cpus. (Admin may view all cpus avaliable without filter by price or other.)
http://localhost:3000/:cpuBrand GET Retrieve all cpus in cpus table that share the same brand. (Admin may want to view all cpus by brand.)
http://localhost:port/cpuBrand/price/coreCount/:serialNumber GET Retrieve cpus based on serial number. (In case Admin needs to check that there are no errors in having duplicated serial numbers.)
http://localhost:3000/:cpuBrand/price/coreCount/:serialNumber GET Retrieve cpus by brand and serial number. (In case Admin needs to check that there are no errors in having duplicated serial numbers.)
http://localhost:3000/ POST Post a listing with no specified parameters (Admin is intended to have easy post ability.)
http://localhost:3000/:serialNumber/ PUT Update an existing cpu by serialNumber (Admin only needs serial number to confirm the listing because Admin has trusted role.)
http://localhost:3000/:serialNumber/ DELETE Delete a cpu by serial number (Admin only needs serial number to confirm the listing.)
